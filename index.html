Agentic AI Code Debugger

An autonomous debugging agent that goes beyond a chat wrapper — multi-step reasoning, tool use to read, run, and trace code, and persistent context memory across sessions. The kind of assistant that gets better at your codebase the longer it works on it.

PythonLangChainOpenAI APIFAISSRAG
The problem
Most "AI debuggers" forget your codebase the moment the chat closes. Context gets lost. Recurring errors get re-debugged from scratch.
The approach
A FAISS-backed memory layer indexes project files, prior fixes, and recurring error traces. RAG retrieves task-relevant context per query. The agent reads, runs, and traces code through tool use.
Status
Core agent loop and memory layer built. Targeting CLI front-end integration with local repos. Code on GitHub.

05 — Principles
01

Ship something, then ship something better.

Done beats perfect. A working v1 in production teaches you more about a problem than three weeks of planning. The improvement loop is where the craft lives.

02

Not a thin wrapper.

If the value of my project is "I called the OpenAI API," I haven't built anything. Real work is the memory layer, the retrieval strategy, the evaluation — the things the API can't do on its own.

03

Read the docs first.

Stack Overflow is fine. The actual library documentation is usually faster, more correct, and respects your time. Start there. Search second.

04

Commit messages are for humans.

"Fixed login bug where capital letters in email broke auth" beats "fix stuff." Future-me, code reviewers, and the person inheriting the repo are all the same person: someone trying to understand what past-me was thinking.
